# Galaxy View — Obsidian 电影感 3D 图谱插件 · 实施计划

## Context（背景与目标）

Rick 的 vault「Rick's Second Brain」实测规模：**3,225 篇笔记、约 19,500 条去重有效边**（注意：当前 2D 图谱开了「显示未解析链接」，实际渲染约 9,773 节点 / 27,800 边）。自带 2D Graph View 在这个规模下既不惊艳、也难以发现关联 pattern。

目标：新建 Obsidian 插件 **Galaxy View**（id: `galaxy-view`，中文名「星系视图」），提供 NASA "Eyes on Asteroids" 风格的电影感 3D 图谱——对 PKM 而言，颜值和可玩性本身就是生产力。

### 已确认的产品决策（Rick 拍板，2026-06-12）
- **V1 只做全局图谱**，打磨到惊艳；局部图放 V2
- **未来上架社区商店** → 第一天起按官方规范开发
- **桌面 + 移动**（vault 经 iCloud 同步到 iOS），移动端降级档位；预先约定退路：若一周调优达不到底线，首发 `isDesktopOnly: true`，移动端进点版本
- **可玩性四件套全要**：搜索+镜头飞行、文件夹/标签着色、节点预览卡片、闲置巡航

## 调研结论摘要（5 个并行调研 agent，已核实，2026-06-12）

- **竞品全军覆没**：4 个已发布 3D 插件全是 `3d-force-graph` 薄封装、全部 MIT、全部停更；死因是逐对象渲染的性能墙（"3D Graph New" 作者原话 "performance issue that I don't know how to fix"）。没有一个做了 bloom/电影镜头/聚合渲染。**不 fork，全新构建**；MIT 竞品作参考（Apoo711 看集成、Hananoshika 看搜索聚焦、TagsRoutes 看粒子）。
- **NASA Eyes 配方（直接读其生产 bundle 验证）**：three.js r167 + UnrealBloomPass（强度=用户滑杆）+ 自定义星空 shader + **DOM 浮层标签** + Tween 缓动镜头 + 细低饱和轨迹线；无雾、无暗角、无颗粒。**高级感 = 克制**。
- **三个 Plan agent 设计视角**（架构/视觉/风险）完整产出已存于 `/tmp/galaxy-design/{architecture,visual,risk}.md` —— **M0 第一件事把它们拷入仓库 `docs/design/`**（/tmp 重启即失；备份源：`/private/tmp/claude-501/-Users-rick-Claude-Code-Obsidian-PKM/2cf2fa95-a0a1-4de2-b146-9705849eef55/tasks/wzhwurhzm.output`）。

依赖方向严格：`data/settings/theme/quality` 不知道 three.js；`layout/render` 不知道 Obsidian；`GraphController` 是唯一全知者。**只有 LayoutEngine/GraphRenderer 这一对接口**——不是过度抽象，是对前人死因的设防（第二用例已由四具尸体付费）。

### 数据层
- 节点：path(主键)/title/folder(TFile.parent)/tags(getAllTags)/degree/ctime/mtime/isUnresolved/isAttachment；边：src/dst/count。
- **V1 默认只渲染有效链接**（3,225 n / ~19.5k e）；「显示未解析」（+6,548 幽灵节点）做成性能门控开关，由 M0 的 S3 基准决定去留——这是对 graph.json parity 的有意偏离，原因：naive parity 等于复刻前人死亡现场。
- 更新策略：**全量重建 + 身份保持合并**（Map<path,node> 原地复用对象引用 → 保住 x/y/z，布局不爆炸）；800ms 尾随 debounce 挂 `metadataCache.on('resolved'/'changed'/'deleted')` + `vault.on('rename'/'delete')`（rename 不被 'changed' 覆盖）。Rick 装有 readwise/cubox 批量同步插件——M1 验证项含真实 Cubox 同步风暴测试。
- 暖启动：插件 data.json 缓存沉降后的坐标，未变化的 vault 重开 <1.5s 直接呈现。

### graph.json 继承（一次性导入，try/catch，永不回写）
Rick 已配 9 个按文件夹的 colorGroups（颜色为十进制 int，query 带尾随空格——解析必须 trim + 防御）。映射表：colorGroups→分组色（int→hex）；nodeSizeMultiplier(0.93) 直通；repelStrength(14.8)→charge≈-14.8×12、linkDistance(264)→×0.3（2D/3D 力空间不可通约，校准常数在预览门调）；showTags/showAttachments/hideUnresolved/showOrphans→过滤器。失败静默回退到内置调色板。

### 生命周期（前人头号 bug 是泄漏，这里是合同）
- onOpen 只搭 DOM 骨架；WebGL 初始化推迟到首个非零尺寸 onResize（避开 deferred/恢复布局的 0×0 bug）。
- RAF 仅在 document 可见 + leaf 实际显示（IntersectionObserver + 'layout-change'）时运行。
- **onClose 销毁清单（按序）**：定时器 → Overlay DOM → worker.terminate + revokeObjectURL → pauseAnimation → bloom pass dispose → 场景遍历 dispose 几何/材质/纹理 → controls.dispose → renderer.dispose + forceContextLoss → ForceGraph3D._destructor() → 引用置空。
- `webglcontextlost` → 「渲染上下文丢失，点击重建」覆盖按钮。
- **泄漏金丝雀**：调试命令「开关视图×10 测内存」，通过线 = 堆增量 <20MB 且无 context 警告，**每个里程碑必跑**。

## 视觉规格（出货目标 = AggregateRenderer）

**核心裁决（三视角分歧的整合）**：聚合渲染不是应急预案，是 V1 出货目标——因为「发光球体」的 NASA 观感本来就要自定义 radial-falloff shader，而它同时把 ~22.7k draw call 压到 **<45 draw call**，性能与美学在此汇合。3d-force-graph 保留为布局/相机/控制宿主。M0 用库默认管线先测真实数字（见里程碑）。

### 场景构成（方向 A 基线）
- 背景 `#000003`（近黑非纯黑，bloom 衰减需要）；无雾无暗角无颗粒。
- **星空**：3 个尺寸级 THREE.Points 共 3 draw call（2600/900/250 点，球壳分布半径 6.5×图半径，85% 冷白 10% 暖白 5% 蓝；~3% 大星 ×1.8 亮度独享 bloom）；整体 0.0008 rad/s 缓转。
- **节点 = 1×THREE.Points + 自定义 shader**（白热核心 smoothstep + 软边缘；per-vertex color/size/dimFactor）。尺寸 `2.2×(1+0.5√degree)` clamp 6 倍（概念词典 degree=689 不能吞掉画面）×用户的 0.93 系数。**邻近升格池**：32 个 Sprite，投影半径 >48px 的节点从 Points 升格（GPU 点尺寸上限会让飞近的节点反而变小——这是核心行为不是彩蛋，重点在 Intel 核显上测）。
- **链接 = 1×LineSegments**（~19.5k 边一次画完）：1px、端点组色 50/50 混合后去饱和 60%、**NormalBlending alpha≈0.16**（additive 在枢纽核心会白爆，实测失败模式；留成「链接辉光」开关）；depthWrite:false，节点永远盖住链接网。
- **状态**：hover=共享加色光晕 Sprite（2.6×半径，120ms 淡入）；选中=3.2×光晕+2.4s 呼吸+核心提亮；**聚焦模式**=非邻居 dimFactor→0.12（280ms 缓动），ESC/点空白恢复；选中节点自身链接全饱和 alpha 0.85，其余 0.04。
- 未解析节点=幽灵处理（无热核、alpha×0.45、--graph-node-unresolved 色）；孤儿=正常渲染（力学漂移到外围就是它的视觉身份）。

### 后期链
`RenderPass → UnrealBloomPass(threshold 0.10, strength 0.9【用户滑杆 0–2.5】, radius 0.45) → OutputPass`（three r152+ 必须，否则线性色洗白）。ACES 色调映射 exposure 1.05（若组色偏移在预览门退回 NoToneMapping）。**无 OutlinePass 无 FXAA**。

### 镜头编排（数字已定，实现者无需品味）
- **开场镜头（第一印象=全部）**：Worker 预热布局至 alpha<0.05（或 1.8s 上限），#000003 遮罩 +「构建星图…」脉动文字；揭幕 600ms 内镜头**从图内部拉出**（0.5×→2.2×图半径，仰角+18°，3.2s easeInOutSine），bloom 1.6→0.9 同步回落；直接交棒巡航。任何输入 250ms 切到终帧。仅视图打开时播放。
- **飞行**：目标距离 `clamp(节点半径×12, 40, 140)`；方位偏移 15°（不正对，留出邻域）；时长 `clamp(800+0.45×航程, 800, 1800)ms` easeInOutCubic；可打断；到达即选中。自有 tween 包装（每帧驱动 cameraPosition(pos,lookAt,0)），不依赖库内部 tween。
- **巡航**：绕质心 0.022 rad/s（约 4.8 分钟/圈）；仰角 ±8°(90s) + 半径 ±4%(60s) 两个不可通约周期 → 轨迹永不复现（「飞船感」而非「转盘」）；有选中时改绕选中节点（NASA 追踪小行星感）；任何输入暂停，10s 无输入后 2s 缓升恢复。

### 两个视觉方向（按 Rick 协议：跑起来看着选，G2 门）
两套 token 全部集中在 `src/render/presets.ts`，运行时命令「切换视觉方案」即时 A/B：
- **A「深空」**：始终深色电影感，无视 app 主题（像视频播放器的刻意反差）；UI 为暗玻璃（blur 16px）。
- **B「随主题」**：深色 = 共用 A 场景；浅色 = **「晨昼制图室」专门设计**——暖纸底 `#F6F4EF`、600 个尘埃微粒替代星空、**bloom 关**（亮底辉光=雾霾）、墨水圆盘节点（L 60%→44% 重定向 9 色相）+1px 深 rim+接触阴影、铅笔线链接（选中时彩墨全饱和——亮色模式的高光时刻）。css-change 切换：180ms 淡出→换 token→220ms 淡入，无 WebGL 重建。
- 落选方向保留在 flag 后面（成本≈一个 token 对象），这是「默认值之门」不是「删除之门」。

### 标签与卡片（DOM 浮层，硬预算 40 个元素）
选中 1 + hover 1 + 邻居 ≤20 + 常驻枢纽标签 top-14（按 degree）；每帧 Vector3.project → translate3d（GPU 合成无重排）；两档字号 11px/13px（连续缩放会抖，禁止）；远距仅最亮的「星座名」浮现。**选中卡片**：标题/带组色点的路径/标签 chips≤5/「↩N 反链 · →M 出链 · 修改于…」/异步 120 字摘要（vault.cachedRead，可取消）/「打开笔记」(workspace.openLinkText) +「聚焦」。**全部 createEl()，禁 innerHTML**（商店审核红线）。移动端换底部抽屉（40vh，下滑关闭）。
